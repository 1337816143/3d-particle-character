# 🎄 3D粒子人物可视化 - 完整部署教程

## 📦 你需要准备的文件

1. `project_3d_3_final.html` - 主HTML文件（只需发送这一个给朋友）
2. `config.json` - 配置文件（包含照片、音乐、书信等）
3. `model.glb` - 3D模型文件

## 🚀 快速开始（3步部署）

### 第1步：创建GitHub仓库

1. 访问 [GitHub](https://github.com) 并登录
2. 点击右上角 `+` → `New repository`
3. 仓库名称：`3d-particle-character`（或任意名称）
4. 选择 `Public`（公开）
5. 点击 `Create repository`

### 第2步：上传文件

1. 在仓库页面点击 `Add file` → `Upload files`
2. 拖拽以下3个文件到上传区域：
   - `project_3d_3_final.html`
   - `config.json`
   - `model.glb`
3. 在底部输入提交信息：`Initial commit`
4. 点击 `Commit changes`

### 第3步：启用GitHub Pages

1. 进入仓库页面
2. 点击 `Settings`（仓库设置）
3. 左侧菜单找到 `Pages`
4. 在 `Build and deployment` 下：
   - `Source` 选择 `Deploy from a branch`
   - `Branch` 选择 `main`
   - 点击 `Save`

5. 等待1-2分钟，页面会显示你的网站URL：
   ```
   https://YOUR_USERNAME.github.io/3d-particle-character/project_3d_3_final.html
   ```

### 🔄 更新内容（无需修改HTML）

#### 更新照片：
1. 在GitHub仓库中找到 `config.json`
2. 点击文件名进入编辑页面
3. 点击右上角 `✎`（铅笔图标）
4. 修改 `photos` 数组，添加照片URL：
   ```json
   {
     "photos": [
       "https://example.com/photo1.jpg",
       "https://example.com/photo2.jpg"
     ]
   }
   ```
5. 点击 `Commit changes`

#### 更新音乐：
修改 `musicUrl` 字段：
```json
{
  "musicUrl": "https://example.com/music.mp3"
}
```

#### 更新书信：
修改 `letter` 字段：
```json
{
  "letter": "新的书信内容..."
}
```

#### 更换主题：
修改 `theme` 字段：
```json
{
  "theme": 0  // 0=金色主题, 1=冰雪主题
}
```

## 📸 如何上传照片到云端

### 方法1：使用图床服务（推荐）

#### ImgBB（免费，无限存储）
1. 访问 [ImgBB](https://imgbb.com)
2. 上传你的照片
3. 获取直接链接（Direct Link）
4. 复制URL到 `config.json` 的 `photos` 数组

#### Cloudinary（免费，25GB存储）
1. 访问 [Cloudinary](https://cloudinary.com)
2. 注册账号
3. 上传照片
4. 复制URL到 `config.json`

#### GitHub仓库本身（最简单）
1. 在仓库中创建 `photos` 文件夹
2. 上传照片到该文件夹
3. 使用GitHub原始URL：
   ```
   https://raw.githubusercontent.com/YOUR_USERNAME/3d-particle-character/main/photos/photo1.jpg
   ```

### 方法2：使用免费CDN

#### Cloudflare R2（免费10GB）
1. 注册 [Cloudflare R2](https://dash.cloudflare.com/)
2. 创建存储桶
3. 上传文件
4. 获取公开访问URL

#### Supabase Storage（免费500MB）
1. 注册 [Supabase](https://supabase.com)
2. 创建项目
3. 在Storage中上传文件
4. 获取公开URL

## 🔧 config.json 完整配置说明

```json
{
  "modelName": "model.glb",
  "modelUrl": "https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_REPO/main/model.glb",
  "photos": [
    "https://example.com/photo1.jpg",
    "https://example.com/photo2.jpg"
  ],
  "musicUrl": "https://example.com/music.mp3",
  "letter": "在这个特别的时刻，\n我想告诉你，\n你是我眼中的万千星河。\n\n(请点击右上角上传书信修改此内容)",
  "theme": 0
}
```

### 字段说明：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|--------|
| `modelName` | string | 模型文件名 | `"model.glb"` |
| `modelUrl` | string | 模型的完整URL | `"https://raw.githubusercontent.com/..."` |
| `photos` | array | 照片URL数组 | `["url1", "url2"]` |
| `musicUrl` | string | 音乐URL | `"https://example.com/music.mp3"` |
| `letter` | string | 书信内容（支持`\n`换行） | `"你好\n世界"` |
| `theme` | number | 主题：0=金色，1=冰雪 | `0` 或 `1` |

## 📝 高级技巧

### 1. 使用多个配置文件
创建不同配置文件：
- `config-birthday.json` - 生日版
- `config-christmas.json` - 圣诞版
- `config-valentine.json` - 情人节版

修改HTML中的 `STATE.configUrl` 来切换：
```javascript
STATE.configUrl = 'config-birthday.json';
```

### 2. 自动更新检测
添加定时刷新配置（可选）：
```javascript
setInterval(async () => {
    await loadConfig();
    console.log('配置已更新');
}, 60000); // 每60秒检查一次
```

### 3. 使用GitHub Actions自动部署
创建 `.github/workflows/deploy.yml` 实现自动部署（高级）

## ❓ 常见问题

### Q: config.json修改后多久生效？
A: 修改后立即生效！朋友刷新页面即可看到最新内容。

### Q: 照片太大怎么办？
A: 建议压缩照片到1MB以下，使用 [TinyPNG](https://tinypng.com) 或 [Squoosh](https://squoosh.app)。

### Q: 音乐文件太大怎么办？
A: 建议使用mp3格式，比特率128kbps，文件大小控制在5MB以下。

### Q: 可以使用其他托管服务吗？
A: 可以！任何支持HTTPS的托管服务都可以，只需确保URL可公开访问。

### Q: GitHub Pages有流量限制吗？
A: 有软限制：100GB带宽/月。对于个人使用完全足够。
